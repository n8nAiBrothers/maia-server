# Skill: Product Manager (Criar PRD)

Sempre que o usuário solicitar o planejamento de uma nova feature, app ou produto, você deve gerar um **Product Requirements Document (PRD)** no melhor padrão de mercado.

## Passos para Execução:

1. **Visão Geral**:
   - Defina o Problema e a Solução proposta.
   - Liste os Objetivos de Negócio (KPIs esperados) e o Público-Alvo.

2. **Requisitos Funcionais e Não Funcionais**:
   - Descreva as features essenciais (Must-have, Should-have, Could-have).
   - Defina requisitos de SEO, Acessibilidade, Performance e Segurança.

3. **Arquitetura e Fluxo (User Journey)**:
   - Descreva o passo a passo do usuário.
   - Utilize diagramas Mermaid (opcional) para ilustrar a arquitetura ou o fluxo de dados.

4. **SEO e GEO (Padrão OBRIGATÓRIO)**:
   - Todo PRD de web app/site DEVE incluir uma seção detalhando a estratégia de SEO (Meta tags, sitemaps, estrutura de headings) e GEO (internacionalização, se aplicável).

5. **Entregável**:
   - Gere o PRD em um arquivo Markdown bem formatado (ex: `PRD_NomeDoProjeto.md`) e salve no projeto.
